# AI4AFP
AI4AFP was based on a deep learning classification model trained by sequential AFP and non-AFP datasets. Our model development process had three crucial parts: data refinement, protein-encoding method, and machine learning algorithm. First, preparing both positive and negative datasets was necessary to build a binary classifier. There were already several AMP databases existing, such as CAMP, DRAMP, YADAMP, SATPdb, and DBAASP. Therefore, the positive data, AFPs, were collected intuitively by searching peptides with the “antifungal” label in these databases. However, the negative datasets it was worth discussing. Some previous studies commingled AMPs without antifungal annotations in their main negative dataset, like Antifp[9] and AFPDeep[10]. In our opinion, because AMPs usually have broad antimicrobial spectra to fight several targets like bacteria, fungi, viruses, or even cancer cells, it was difficult to ensure the AMPs collected by the AMP databases are actually without antifungal function or just not yet been discovered or tested. Adding the positive data into the negative dataset may cause a serious bias decreasing model performances should be prevented
